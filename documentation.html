1. Introduction et objectifs du projet

Le Reverse Proxy Manager est une solution auto-hebergee permettant la gestion dynamique de regles de reverse proxy Nginx via une interface web intuitive. L'ensemble fonctionne dans un unique conteneur Docker.

Objectifs

  • Simplifier la configuration de reverse proxy sans editer manuellement les fichiers Nginx.
  • Centraliser la gestion des redirections dans une interface unique accessible via HTTPS.
  • Presenter les services disponibles sous forme de tuiles cliquables sur une page d'accueil publique.
  • Securiser l'administration par mot de passe avec protection anti-brute-force.
  • Automatiser la validation et le rechargement de Nginx apres chaque modification.
  • Persister les configurations entre les redemarrages grace a un volume Docker dedie.

6. Configuration Nginx

6.1 Configuration principale

user nginx;
worker_processes auto;
error_log /var/log/nginx/error.log warn;
pid /var/run/nginx.pid;

events { worker_connections 1024; }

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;
    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent"';
    access_log /var/log/nginx/access.log main;
    sendfile on;
    keepalive_timeout 65;

    # Administration + Accueil (HTTPS, default)
    server {
        listen 443 ssl default_server;
        server_name _;
        ssl_certificate /etc/nginx/certs/selfsigned.crt;
        ssl_certificate_key /etc/nginx/certs/selfsigned.key;
        ssl_protocols TLSv1.2 TLSv1.3;

        location / {
            proxy_pass http://127.0.0.1:5000;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }

    # Redirect HTTP -> HTTPS
    server {
        listen 80 default_server;
        server_name _;
        return 301 https://$host$request_uri;
    }

    # Routes dynamiques
    include /etc/nginx/conf.d/*.conf;
}

6.2 Generation dynamique des routes

Chaque route cree un bloc server {} dans /etc/nginx/conf.d/proxy.conf avec support WebSocket natif :

server {
    listen 443 ssl;
    server_name app.exemple.com;
    ssl_certificate /etc/nginx/certs/selfsigned.crt;
    ssl_certificate_key /etc/nginx/certs/selfsigned.key;

    location / {
        proxy_pass http://192.168.1.50:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
Port 80 : Si une route utilise le port 80, les directives SSL sont automatiquement omises pour eviter les conflits avec le bloc de redirection HTTP→HTTPS.

7.4 Gestion des tuiles

Schema d'une tuile

ChampTypeObligatoireDescription
idstring (8 hex)AutoIdentifiant unique genere automatiquement
namestringOuiNom affiche sur la tuile
urlstring (URL)OuiURL de redirection au clic

logostring (filename)NonNom du fichier image uploade

Formats d'image acceptes

PNG, JPG/JPEG, GIF, SVG, WebP, ICO

Cycle de vie d'un logo

  1. L'image est uploadee via le formulaire (enctype multipart/form-data).
  2. Le fichier est renomme avec un UUID unique pour eviter les collisions.
  3. Il est stocke dans /data/logos/ (volume persistent).
  4. Lors de la modification, l'ancien logo est supprime si un nouveau est uploade.
  5. Lors de la suppression d'une tuile, le logo associe est supprime.

12. Guide d'utilisation

12.1 Gerer les tuiles

Ajouter une tuile

  1. Connectez-vous sur /admin.
  2. Dans "Gestion des tuiles", remplissez :
ChampExempleDescription
NomNextcloudNom affiche sur la tuile
URL de redirectionhttps://cloud.mondomaine.frLien ouvert au clic (nouvel onglet)
Logonextcloud.pngImage optionnelle (PNG, JPG, SVG...)
Position1Ordre d'affichage (0 = premier)
  1. Cliquez sur Ajouter. La tuile apparait immediatement sur la page d'accueil.

Modifier / Supprimer une tuile

  • Modifier : Bouton "Modifier" → edition en ligne du nom, URL et position. Cliquer "Sauver".
  • Supprimer : Bouton "Supprimer" avec confirmation. Le logo est automatiquement supprime.

12.2 Gerer les routes proxy

Ajouter une route

ChampExempleDescription
Domaineapp.mondomaine.frserver_name Nginx
Ciblehttp://192.168.1.50:3000URL du backend
Port443Port d'ecoute (defaut 443)
Rappel : Les tuiles et les routes proxy sont independantes. Une tuile peut pointer vers n'importe quelle URL, pas necessairement un service proxye.
