1. Introduction et objectifs du projet

Le Reverse Proxy Manager est une solution auto-hebergee permettant la gestion dynamique de regles de reverse proxy Nginx via une interface web intuitive. L'ensemble de l'application fonctionne dans un unique conteneur Docker, combinant un serveur Nginx performant et une application d'administration Flask.

Objectifs

  • Simplifier la configuration de reverse proxy sans editer manuellement les fichiers Nginx.
  • Centraliser la gestion des redirections dans une interface unique accessible via HTTPS.
  • Automatiser la validation et le rechargement de Nginx apres chaque modification.
  • Persister les configurations entre les redemarrages grace a un volume Docker dedie.
  • Securiser les communications avec TLS 1.2/1.3 par defaut.

Cas d'usage

Homelab / Self-hostingExposer plusieurs services internes (Nextcloud, Gitea, Jellyfin...) derriere un point d'entree unique.

4.2 Generation des certificats SSL

Avant le premier demarrage, generez les certificats auto-signes :

# Rendre le script executable
chmod +x gen-certs.sh

# Generer les certificats
bash gen-certs.sh

Le script effectue les operations suivantes :

  1. Cree le repertoire certs/ s'il n'existe pas.
  2. Verifie si des certificats existent deja (idempotent).
  3. Genere une paire cle/certificat RSA 2048 bits, valide 10 ans.
# Commande OpenSSL executee par le script :
openssl req -x509 -nodes -days 3650 \
    -newkey rsa:2048 \
    -keyout certs/selfsigned.key \
    -out certs/selfsigned.crt \
    -subj "/C=FR/ST=Local/L=Local/O=ReverseProxy/CN=localhost"

# Copy application
COPY app/ /app/

# Copy nginx config
COPY nginx/nginx.conf /etc/nginx/nginx.conf

# Create directories

# Create empty proxy.conf

6. Configuration Nginx

6.1 Configuration principale

user nginx;
worker_processes auto;
error_log /var/log/nginx/error.log warn;
pid /var/run/nginx.pid;

events {
    worker_connections 1024;
}

http {
    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent" "$http_x_forwarded_for"';

    access_log /var/log/nginx/access.log main;
    sendfile on;
    keepalive_timeout 65;

    # Panel web d'administration (HTTPS)
    server {
        listen 443 ssl default_server;
        server_name _;

        ssl_certificate /etc/nginx/certs/selfsigned.crt;
        ssl_certificate_key /etc/nginx/certs/selfsigned.key;
        ssl_protocols TLSv1.2 TLSv1.3;

        location / {
            proxy_pass http://127.0.0.1:5000;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }
    }

    # Redirect HTTP -> HTTPS
    server {
        listen 80 default_server;
        server_name _;
        return 301 https://$host$request_uri;
    }

    # Include dynamic proxy configs
    include /etc/nginx/conf.d/*.conf;
}

6.2 Generation dynamique des routes

Chaque route configuree via l'interface web genere un bloc server {} dans le fichier /etc/nginx/conf.d/proxy.conf. Ce fichier est entierement regenere a chaque modification.

Exemple de configuration generee

Pour une route : domaine app.exemple.com → cible http://192.168.1.50:3000 sur le port 443 :

server {
    listen 443 ssl;
    server_name app.exemple.com;

    ssl_certificate /etc/nginx/certs/selfsigned.crt;
    ssl_certificate_key /etc/nginx/certs/selfsigned.key;

    location / {
        proxy_pass http://192.168.1.50:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
Support WebSocket : Chaque route generee inclut automatiquement les en-tetes Upgrade et Connection pour le support natif des WebSockets.

Cycle de vie d'une modification

  1. Reception : L'utilisateur soumet le formulaire (POST).
  2. Validation : Verification des champs obligatoires (domaine, cible). Ajout automatique du prefixe http:// si absent.
  3. Persistance : Mise a jour de /data/routes.json.
  4. Generation : Regeneration complete de /etc/nginx/conf.d/proxy.conf.
  5. Validation Nginx : Execution de nginx -t pour verifier la syntaxe.
  6. Rechargement : Si valide, execution de nginx -s reload (zero downtime).
  7. Notification : Message flash affiche a l'utilisateur (succes ou erreur).

8. Gestion des processus (Supervisor)

Supervisor est le processus principal (PID 1) du conteneur. Il gere le cycle de vie de deux processus enfants : Nginx et Flask/Gunicorn. 

[supervisord]
nodaemon=true
logfile=/var/log/supervisord.log
pidfile=/var/run/supervisord.pid

[program:nginx]
command=nginx -g "daemon off;"
autostart=true
autorestart=true
stdout_logfile=/dev/stdout
stdout_logfile_maxbytes=0
stderr_logfile=/dev/stderr
stderr_logfile_maxbytes=0

[program:flask]
command=gunicorn -w 2 -b 127.0.0.1:5000 app:app
directory=/app
autostart=true
autorestart=true
stdout_logfile=/dev/stdout
stdout_logfile_maxbytes=0
stderr_logfile=/dev/stderr
stderr_logfile_maxbytes=0

Detail des processus supervises

ProgrammeCommandeRedemarrage autoLogs
nginxnginx -g "daemon off;"Ouistdout/stderr → Docker logs
flaskgunicorn -w 2 -b 127.0.0.1:5000 app:appOuistdout/stderr → Docker logs

Parametres Gunicorn

ParametreValeurDescription
-w 22 workersNombre de processus de travail (adapte au panneau d'admin)
-b 127.0.0.1:5000localhost:5000Ecoute locale uniquement (securite)

Pourquoi nodaemon=true ? Docker attend que le processus PID 1 reste au premier plan. Sans cette option, Supervisor se daemoniserait et le conteneur s'arreterait immediatement.

10. Guide d'utilisation

10.1 Ajouter une route

  1. Accedez au panneau d'administration : https://<IP-du-serveur>
  2. Dans le formulaire "Ajouter une redirection" :
ChampExempleDescription
Domaine / Hostapp.mondomaine.frLe nom de domaine que Nginx ecoutera
Cible (IP:port ou URL)http://192.168.1.50:3000URL du service backend (prefixe http:// ajoute automatiquement)
Port d'ecoute443Port sur lequel Nginx ecoute pour cette route (defaut : 443)
  1. Cliquez sur "Ajouter".
  2. Un message de confirmation s'affiche. Nginx est automatiquement recharge.
Astuce : Si vous specifiez un port different de 443, pensez a l'exposer dans le docker-compose.yml (section ports) et a redemarrer le conteneur.

10.2 Modifier une route

  1. Dans le tableau des routes, cliquez sur le bouton "Modifier" de la route souhaitee.
  2. Les cellules du tableau deviennent editables (champs de saisie en ligne).
  3. Modifiez les valeurs souhaitees (domaine, cible, port).
  4. Cliquez sur "Sauver" pour appliquer les modifications.
  5. Cliquez sur "Annuler" pour abandonner les modifications.

10.3 Supprimer une route

  1. Dans le tableau des routes, cliquez sur le bouton "Supprimer" de la route souhaitee.
  2. Une boite de dialogue de confirmation apparait : "Supprimer cette route ?".
  3. Confirmez pour supprimer. Nginx est automatiquement recharge.
Attention : La suppression est immediate et irreversible. Aucun historique des modifications n'est conserve. Pensez a sauvegarder le fichier routes.json regulierement.

Annexe A — Code source complet : app.py

import json
import os
import subprocess
from flask import Flask, render_template, request, redirect, url_for, flash

app = Flask(__name__)
app.secret_key = os.environ.get("SECRET_KEY", "change-me-in-production")

CONFIG_FILE = "/data/routes.json"


def load_routes():
    if os.path.exists(CONFIG_FILE):
        with open(CONFIG_FILE) as f:
            return json.load(f)
    return []


def save_routes(routes):
    os.makedirs(os.path.dirname(CONFIG_FILE), exist_ok=True)
    with open(CONFIG_FILE, "w") as f:
        json.dump(routes, f, indent=2)


def generate_nginx_conf(routes):
    """Generate nginx upstream config from routes."""
    conf_path = "/etc/nginx/conf.d/proxy.conf"
    lines = []
    for route in routes:
        domain = route["domain"]
        target = route["target"]
        listen_port = route.get("listen_port", "443")
        lines.append(f"""server {{
    listen {listen_port} ssl;
    server_name {domain};

    ssl_certificate /etc/nginx/certs/selfsigned.crt;
    ssl_certificate_key /etc/nginx/certs/selfsigned.key;

    location / {{
        proxy_pass {target};
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }}
}}
""")
    with open(conf_path, "w") as f:
        f.write("\n".join(lines) if lines else "# No routes configured\n")


def reload_nginx():
    try:
        subprocess.run(["nginx", "-t"], check=True,
                       capture_output=True, text=True)
        subprocess.run(["nginx", "-s", "reload"], check=True,
                       capture_output=True, text=True)
        return True, "Nginx recharge avec succes."
    except subprocess.CalledProcessError as e:
        return False, f"Erreur Nginx: {e.stderr}"


@app.route("/")
def index():
    routes = load_routes()
    return render_template("index.html", routes=routes)


@app.route("/add", methods=["POST"])
def add_route():
    domain = request.form.get("domain", "").strip()
    target = request.form.get("target", "").strip()
    listen_port = request.form.get("listen_port", "443").strip()

    if not domain or not target:
        flash("Le domaine et la cible sont obligatoires.", "error")
        return redirect(url_for("index"))

    if not target.startswith(("http://", "https://")):
        target = "http://" + target

    routes = load_routes()
    routes.append({
        "domain": domain,
        "target": target,
        "listen_port": listen_port,
    })
    save_routes(routes)
    generate_nginx_conf(routes)
    ok, msg = reload_nginx()
    flash(msg, "success" if ok else "error")
    return redirect(url_for("index"))


@app.route("/delete/<int:route_id>", methods=["POST"])
def delete_route(route_id):
    routes = load_routes()
    if 0 <= route_id < len(routes):
        removed = routes.pop(route_id)
        save_routes(routes)
        generate_nginx_conf(routes)
        ok, msg = reload_nginx()
        flash(f"Route {removed['domain']} supprimee. {msg}",
              "success" if ok else "error")
    else:
        flash("Route introuvable.", "error")
    return redirect(url_for("index"))


@app.route("/edit/<int:route_id>", methods=["POST"])
def edit_route(route_id):
    routes = load_routes()
    if 0 <= route_id < len(routes):
        domain = request.form.get("domain", "").strip()
        target = request.form.get("target", "").strip()
        listen_port = request.form.get("listen_port", "443").strip()

        if not domain or not target:
            flash("Le domaine et la cible sont obligatoires.", "error")
            return redirect(url_for("index"))

        if not target.startswith(("http://", "https://")):
            target = "http://" + target

        routes[route_id] = {
            "domain": domain,
            "target": target,
            "listen_port": listen_port,
        }
        save_routes(routes)
        generate_nginx_conf(routes)
        ok, msg = reload_nginx()
        flash(msg, "success" if ok else "error")
    else:
        flash("Route introuvable.", "error")
    return redirect(url_for("index"))


if __name__ == "__main__":
    generate_nginx_conf(load_routes())
    app.run(host="0.0.0.0", port=5000)

C.5 — gen-certs.sh

#!/bin/bash
set -e
CERT_DIR="./certs"
mkdir -p "$CERT_DIR"

if [ -f "$CERT_DIR/selfsigned.crt" ] && [ -f "$CERT_DIR/selfsigned.key" ]; then
    echo "Certificats SSL deja presents dans $CERT_DIR/, rien a faire."
    exit 0
fi

echo "Generation des certificats SSL auto-signes..."
openssl req -x509 -nodes -days 3650 \
    -newkey rsa:2048 \
    -keyout "$CERT_DIR/selfsigned.key" \
    -out "$CERT_DIR/selfsigned.crt" \
    -subj "/C=FR/ST=Local/L=Local/O=ReverseProxy/CN=localhost"

echo "Certificats generes dans $CERT_DIR/"

Annexe E — Glossaire

TermeDefinition
Reverse ProxyServeur intermediaire qui recoit les requetes des clients et les transmet aux serveurs backend. Il masque l'architecture interne et centralise la terminaison TLS.
NginxServeur web haute performance utilise ici comme reverse proxy et terminateur TLS.
FlaskMicro-framework web Python utilise pour l'interface d'administration.
GunicornServeur WSGI Python qui execute l'application Flask en mode production avec plusieurs workers.
SupervisorSysteme de controle de processus qui gere et surveille plusieurs services au sein d'un meme conteneur.
TLS (Transport Layer Security)Protocole cryptographique assurant la confidentialite et l'integrite des communications reseau.
Certificat auto-signeCertificat X.509 signe par sa propre cle privee, sans autorite de certification tierce.
WSGIWeb Server Gateway Interface — standard Python pour la communication entre serveurs web et applications.
proxy_passDirective Nginx qui transmet une requete a un serveur backend specifie.
server_nameDirective Nginx qui determine quel bloc serveur traite une requete en fonction du nom d'hote.
WebSocketProtocole de communication bidirectionnelle en temps reel sur une seule connexion TCP.
Volume DockerMecanisme de persistance des donnees au-dela du cycle de vie d'un conteneur.
Alpine LinuxDistribution Linux ultra-legere (~5 Mo) couramment utilisee comme base pour les images Docker.
